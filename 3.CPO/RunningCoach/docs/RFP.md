## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | RunningCoach |
| 目的 | GARMINデータと練習テーマのPDCAにより、マラソンPB更新を支援するWebアプリ |
| 開発フェーズ | フェーズ1：個人利用ツール → フェーズ3：一般公開SaaS |
| 参照プロジェクト | AX-Diagnosis（同技術スタック） |

---

## 2. 背景・解決したい課題

ランニングの記録向上には「意図を持った練習」と「その振り返り」が不可欠だが、現状は以下の課題がある。

- GARMINで豊富なデータが取得できるが、振り返りに活かしきれていない
- 「今日の練習テーマ」を設定しても、達成できたか客観的に評価する手段がない
- 次の日の練習メニューが目標大会・目標タイムと紐づいていない
- 練習のPDCAが属人的な感覚に頼っており、習慣化・継続が難しい

---

## 3. サービスコンセプト

```
目標大会・目標タイムを起点に、
「テーマ設定 → 実走 → データ取得 → 評価 → 次の練習提案」
のPDCAを毎日回し続けることで、マラソンPB更新をサポートする。
```

## 5. 機能要件

### 5-1. MVP必須機能（フェーズ1）

#### ① 目標・練習計画パネル（`/goal`）

- 目標大会の登録（大会名・開催日・距離）
- 目標タイムの設定（例：3時間30分）
- 目標タイムから逆算した目安ペース・必要トレーニング量の表示
- 週次・月次の練習計画の入力（メニュー種別・距離・強度）
- 大会までの残り日数・練習消化率のダッシュボード表示

#### ② GARMINデータ取込機能（`/log/new`）

**取込対象フィールド（GARMIN CSVより）：**

| カテゴリ | フィールド |
|---------|-----------|
| 基本 | Activity Type / Date / Title / Distance / Time / Calories |
| 心拍 | Avg HR / Max HR / Aerobic TE |
| ペース | Avg Pace / Best Pace / Avg GAP |
| ランニングダイナミクス | Avg Run Cadence / Max Run Cadence / Avg Stride Length / Avg Vertical Ratio / Avg Vertical Oscillation / Avg Ground Contact Time / Avg GCT Balance |
| パワー | Normalized Power® / Avg Power / Max Power / Training Stress Score® |
| コース | Total Ascent / Total Descent / Min/Max Elevation / Moving Time / Elapsed Time |
| 環境・その他 | Min/Max Temp / Avg/Min/Max Resp / Body Battery Drain / Steps |

**取込方法：**
- フェーズ1：CSVファイルの手動アップロード（GARMIN Connectからのエクスポート）
- フェーズ2以降：Garmin Connect API自動連携

#### ③ 計画vs実績の差分評価（`/log/[date]`）

- 練習前に設定したテーマの記録（テキスト入力）
- テーマの種別タグ付け

| テーマ例 | 評価に使うデータ |
|---------|---------------|
| ペースを一定に保つ | Avg Pace / Best Pace / Avg GAP のばらつき |
| LT走で心拍を160台に保つ | Avg HR / Max HR の分布 |
| ケイデンスを180以上にする | Avg Run Cadence / Max Run Cadence |
| 左右バランスを意識する | Avg GCT Balance |
| 接地時間を短くする | Avg Ground Contact Time |
| 上下動を抑える | Avg Vertical Oscillation / Avg Vertical Ratio |

- 自己コメントの入力（自分の感覚・気づき）
- AIによる総合評価コメントの生成（データ＋自己コメントを統合）
- 計画メニューとの差分表示（距離・ペース・負荷）

### 5-2. 翌日練習メニュー提案機能（`/advice`）

**参照する情報（すべて統合して判断）：**

| 情報 | 内容 |
|------|------|
| 目標大会・目標タイム | 最終ゴール・残り期間 |
| 今日の実績データ | 全GARMINフィールド |
| 今日の練習テーマ達成度 | 自動評価＋自己コメント |
| 直近の練習履歴 | 累積疲労・連続日数・負荷推移 |
| 練習計画との進捗差 | 週間計画に対する消化状況 |
| 体調の自己申告 | 疲労感・睡眠・気になる箇所 |
| 過去提案メニューの評価 | 提案→実施→効果のフィードバック |

**提案形式：**
- 理由つきのテキスト提案（「なぜこのメニューか」を明記）
- メニュー内容（距離・強度・ペース目安・心拍ゾーン）
- 翌日以降の週間見通し

**提案メニューへのフィードバック：**
- 提案メニューを実施したかの記録
- 提案に対する評価（良かった / 合わなかった / 別のメニューにした）
- 評価は次回提案に反映

## 8. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブ | スマートフォン・PCどちらでも使用可能 |
| パフォーマンス | AIレスポンスは3秒以内を目安 |

## 9. フェーズ計画

```
フェーズ1（MVP）
  ├ 個人利用
  ├ CSVアップロード
  ├ 目標・計画パネル
  ├ テーマ設定・評価
  ├ AI練習提案（テキスト）
  └ ローカル or Supabaseにデータ保存

フェーズ2
  ├ Garmin Connect API自動連携
  ├ チーム・複数ユーザー対応
  └ 認証（Clerk）

フェーズ3
  ├ 一般公開・SaaS化
  ├ サブスクリプション課金
  └ コーチングモード（指導者が複数選手を管理）
```
